2.3 Post-processing

ElectroacPy provides some tools for crossover-network / filter-design. These are considered “post-processing” as they are applied on computation results. It might not be the best term, but we’ll go with it for now.

2.3.1 .crossover[]

Crossovers/filters are initialized through the .filter_network() method. Input parameters are:

  • name, network reference,
  • ref2bem, which BEM elements should be filtered,
  • ref2study, which study (or list of study) should receive the filter network.

Available filters are:

  • lowpass and highpass, both under their “analog” form and as biquads,
  • low-shelf, high-shelf,
  • peaking eq,
  • delay, gain and phase-flip,
  • user-defined transfer functions.

In the code below, we define a crossover network using the available biquads:

#%% load data
floorstander = ep.load("save/acoustic_radiation")

#%% create LF filters
floorstander.filter_network("LF_filter", [1, 2], "free-field")
floorstander.filter_addLowPassBQ("LF_filter", "lp1", 700, 0.707)
floorstander.filter_addHighShelf("LF_filter", "hs1", 500, 0.707, -16)
floorstander.filter_addPeakEQ("LF_filter", "peq1", 200, 1.2, -2)
floorstander.filter_addDelay("LF_filter", "dt", 90e-6)
floorstander.filter_addGain("LF_filter", "dB", -4)

floorstander.filter_network("HF_filter", 3, "free-field")
floorstander.filter_addHighPassBQ("HF_filter", "hp1", 700, 0.707)

Networks will automatically update the frequency response of previously computed studies. The .plot_results() method will display updated data. If you want to have a better look at each network response, you can either use the .plot_xovers() method, or extract the transfer function with .crossover["reference"].h — Figure 2.14 is made using the following code:

#%% get frequency response
hlf = floorstander.crossover["LF_filter"].h
hhf = floorstander.crossover["HF_filter"].h

#%% plot crossover network
gtb.plot.FRF(floorstander.frequency, (hlf, hhf, hlf+hhf), "dB",
             ylim=(-30, 10), xlim=(10, 10000), figsize=(6, 3),
             xticks=(10, 20, 50, 
                     100, 200, 500, 
                     1000, 2000, 5000, 10000),
             yticks=arange(-30, 12, 6))

It is important to note that electroacPy’s crossover tools are considered as digital filters: interaction between speaker and supposed electrical components is not taken into account. In order to have a better understanding of the electrical behavior with passive components, it is either possible to use the circuitSolver class, or to export results to an external software for crossover design.

2.3.2 Export simulation data

For now, it is only possible to export directivity and impedance data. Because it is free and provides a lot of great tools, we’ll use VituixCAD to build a crossover network. The following code extracts pressure and impedance with .export_directivity() and .export_impedance(). Results are then imported into VituixCAD as shown in Figure 2.17.

#%% extract data
# pressure
floorstander.export_directivity("dir_LF_hor", "free-field", "polar_hor", [1, 2], bypass_xover=True)
floorstander.export_directivity("dir_HF_hor", "free-field", "polar_hor", 3, bypass_xover=True)

# extract 
floorstander.export_impedance("LF_Ze", "ported_LF")
floorstander.export_impedance("HF_Ze", "TW29RN")
